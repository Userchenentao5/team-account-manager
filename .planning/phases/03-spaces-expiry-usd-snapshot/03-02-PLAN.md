<feature>
  <name>Exact BigInt USD freeze (FX-02, D-03)</name>
  <files>src/lib/money.ts, src/lib/money.test.ts</files>
  <behavior>
    freezeUsdMinor(amountMinor: number, srcExp: number, rateToUsd: string, usdExp = 2) → number (USD minor units):
    - USD self: freezeUsdMinor(1999, 2, "1") → 1999 (unchanged; D-02 USD base = 1)
    - 2-decimal source, d == 0: freezeUsdMinor(amountMinor, 2, rate) → round(amountMinor × rate)
    - JPY (srcExp 0 → d = +2): a JPY amount converts without the ~100× corruption (Pitfall 3)
    - Rounding: round-half-up — a remainder where r*2 >= den rounds up
    - No floating-point multiply is used anywhere in the function (CLAUDE.md hard rule)
  </behavior>
  <implementation>
    Add `freezeUsdMinor` to src/lib/money.ts alongside the existing `parseToMinor`/`formatMinor`, in the same float-free, exponent-aware, doc-commented style (RESEARCH Pattern 3, lines 215-235; PATTERNS lines 168-188). Formula: usdMinor = round(amountMinor × rate × 10^(usdExp − srcExp)) computed entirely with BigInt. Split `rateToUsd` on "." into whole+frac, `rateInt = BigInt(whole + frac)`, `den = 10n ** BigInt(frac.length)`; `num = BigInt(Math.abs(amountMinor)) * rateInt`; let `d = usdExp - srcExp`, if d >= 0 multiply num by 10^d else multiply den by 10^(-d); round-half-up via `r * 2n >= den ? q + 1n : q`; restore sign. Follow the existing `throw new Error(...)` guard style (money.ts lines 13-17) for invalid input if you add guards. Extend src/lib/money.test.ts with the cases above; write the new cases FIRST (RED), confirm failure, then implement (GREEN). Do not weaken or remove existing parseToMinor/formatMinor tests.
  </implementation>
</feature>
