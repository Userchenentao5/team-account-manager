<objective>
Build the FX-02 save pipeline as Server Actions in `src/actions/spaces.ts`: `createSpace` and `updateSpace`. Each re-parses input server-side (trust boundary), validates reference data, awaits the async rate refresh, reads the cached rate (blocking if absent — D-02), computes the calendar-aware expiry and the frozen USD amount, and writes via the synchronous 2-table transaction. `updateSpace` honors freeze semantics (re-freeze only on amount/currency change). Integration-tested against the in-memory harness.

Purpose: This is the heart of the phase — FX-02 frozen-snapshot + SPACE-01 create + SPACE-04 edit + ACCT-01 mother, composed from Plan 02 (pure math) and Plan 03 (data + validation) primitives plus the Phase 2 FX service.
Output: `createSpace`/`updateSpace` actions + integration tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client form → Server Action | Untrusted input crosses here; the action is a public endpoint |
| Server Action → SQLite | Parameterized writes; atomic 2-table transaction |
| FX cache → USD freeze | Cached rate feeds money math; empty cache must block, not zero |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-09 | Tampering / Elevation | mass assignment via Server Action | high | mitigate | `spaceFormSchema.safeParse` parses only whitelisted fields; extras stripped — like actions/channels.ts. |
| T-03-10 | Tampering | spoofed inactive channel / unseeded currency | high | mitigate | Server-side reference-data validation against `listChannels(db)` active set + active seeded currencies (Pitfall 6); never trust client select options. |
| T-03-11 | Tampering | poisoned/empty rate into USD math | high | mitigate | D-02 blocks save when no cached rate (never 0/NULL amount_usd); Phase 2 already rejects 0/neg/NaN/∞ at the cache boundary; freeze is BigInt-exact. |
| T-03-12 | Tampering | client RHF/Zod trusted as authority | medium | mitigate | Server Action re-parses; client validation is UX-only. |
| T-03-13 | Integrity | partial write on failure | high | mitigate | Single synchronous `db.transaction` (await done before opening it) — all-or-nothing space+mother insert. |
