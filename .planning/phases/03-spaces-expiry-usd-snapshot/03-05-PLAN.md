## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (client form) → Server Action | Client RHF/Zod is UX only; the real trust boundary is the server re-parse in Plan 04 |
| RSC → SQLite | Read-only list/detail queries on the Node runtime (never Edge) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-14 | Tampering | client-side validation treated as authority | medium | mitigate | Form is UX-only; createSpace/updateSpace re-parse server-side (Plan 04). No security decision is made in the client. |
| T-03-15 | Denial of service / runtime | better-sqlite3 loaded on Edge | medium | mitigate | List + detail routes set `export const dynamic = "force-dynamic"` (Node runtime); never `runtime = "edge"` (Pitfall 4). |
| T-03-16 | Information disclosure | rendering credentials | low | accept | Schema stores only email/login (D-05); there are no credentials to leak in the UI. |
</threat_model>
