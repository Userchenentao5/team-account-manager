# Phase 1 — UI Design Contract

> Visual and interaction contract for the navigation shell + Reference Data screens (payment-channel maintenance and the read-only currency list). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Scope:** single-user internal accounting/asset-management tool. No marketing/landing surfaces, no auth UI. Pragmatic, data-dense admin UI. Light mode is the primary (and only required) theme for Phase 1.
>
> **Stack note:** Greenfield repo — `components.json` does not exist yet. shadcn/ui is LOCKED in `.claude/CLAUDE.md`; `npx shadcn@latest init` runs as part of the Phase 1 scaffold. The values below are the intended configuration the scaffold task must produce, not a re-litigation of the tool choice.

## Spacing Scale

8-point scale. All values are multiples of 4 (Tailwind default scale maps cleanly: `1`=4, `2`=8, `4`=16, `6`=24, `8`=32, `12`=48, `16`=64).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge inner padding |
| sm | 8px | Compact spacing between table action buttons, form field inner gaps |
| md | 16px | Default gap between form fields, dialog content padding, table cell padding-x |
| lg | 24px | Card padding, page content padding, section spacing |
| xl | 32px | Gap between major page sections, page top padding |
| 2xl | 48px | Empty-state vertical padding |
| 3xl | 64px | Reserved for page-level breathing room (not heavily used in Phase 1) |

Exceptions: icon-only action buttons (archive / reactivate / rename triggers) use a **44px minimum touch/click target** (e.g. `size-11` or an `h-9 w-9` button inside a 44px hit area) even though the icon glyph is smaller. This is the only allowed deviation from the 4-multiple block sizing.

---

## Typography

Exactly 4 sizes, exactly 2 weights. Aligned to Tailwind/shadcn defaults (`text-2xl`, `text-lg`, `text-base`, `text-sm`).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Display (page title, e.g. "支付渠道") | 24px | 600 (semibold) | 1.25 |
| Heading (card/section title, dialog title) | 18px | 600 (semibold) | 1.3 |
| Body (table cells, dialog body, descriptions) | 16px | 400 (regular) | 1.5 |
| Label (table column headers, form labels, badges, helper/muted text) | 14px | 600 (semibold) for headers/labels · 400 for muted helper text | 1.4 |

Weights in use: **400 (regular)** and **600 (semibold)** only. No light/medium/bold variants.
Numerics (currency codes, `minor_unit` values) render in Geist Mono at the Body or Label size for column alignment.

## Color

shadcn `neutral` base theme (light mode). 60/30/10 split. A single blue accent is reserved as the brand/primary action color so amber/red remain free for the later dashboard's expiry-status tiers (DASH-01) — keeping the palette consistent across phases.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` (`--background`) | Page background, main content area, table body, dialog surface |
| Secondary (30%) | `#F5F5F5` (neutral-100, `--muted` / `--sidebar`) | Left sidebar, card surfaces, table header row, archived-row background tint |
| Accent (10%) | `#2563EB` (blue-600, `--primary`) | See reserved-for list below |
| Destructive | `#DC2626` (red-600, `--destructive`) | Reserved — **not used in Phase 1** (see note) |

Accent reserved for: **primary CTA button ("新增渠道") fill; the active sidebar nav item (background tint + left indicator + icon); input focus rings; the "show archived" switch when ON; and text links.** Secondary/neutral actions (Cancel, rename trigger, archive trigger, row action icons) use neutral/ghost/outline styling — accent is NOT applied to every interactive element.

Destructive note: Phase 1 has **no irreversible/destructive action**. Channel removal is a *reversible soft-delete* (archive → reactivate, D-06/D-08). The archive confirmation therefore uses a **neutral/secondary** confirm button, not the red destructive style. The destructive token is declared and reserved for the genuinely destructive cascade-delete in Phase 4.

Archived rows: rendered with a muted/secondary background tint and a `Badge` reading "已归档"; the row's primary action flips from "归档" to "恢复".

## Copywriting Contract

UI copy is **Simplified Chinese** to match the product spec and the single Chinese-speaking user. English glosses are for the executor's reference only.

| Element | Copy |
|---------|------|
| Primary CTA | `新增渠道` (Add channel) — opens the add-channel dialog |
| Secondary CTAs | Rename trigger: `重命名` · Archive trigger: `归档` · Reactivate trigger: `恢复` · Dialog confirm: `保存` · Dialog cancel: `取消` |
| Show-archived control | Switch labeled `显示已归档` (Show archived); off by default |
| Empty state heading | `还没有支付渠道` (No payment channels yet) |
| Empty state body | `添加你的第一个支付渠道,之后创建空间时就能选择它。` + inline primary button `新增渠道` |
| Error state (validation) | Empty name: `请输入渠道名称。` · Duplicate active name: `已存在同名的有效渠道。` |
| Error state (server) | Toast (sonner): `保存失败,请重试。` (Save failed, please try again) |
| Success feedback | Toast: add → `已添加渠道` · rename → `已重命名` · archive → `已归档渠道` · reactivate → `已恢复渠道` |
| Archive confirmation | Title: `归档此渠道?` · Body: `归档后,该渠道不会出现在新建空间的选择列表中;已使用它的空间不受影响。你可以随时恢复。` · Confirm button: `归档` (neutral style, not red) · Cancel: `取消` |

**Currency list (REF-02, read-only):** no empty state (6 rows seeded at migration). Table columns: `代码` (code, mono), `名称` (name), `最小单位位数` (minor-unit exponent). A short caption clarifies it is read-only and maintained via seed: `币种列表用于创建空间时选择,暂不支持在此编辑。`

**Navigation shell labels:** `仪表盘` (Dashboard), `空间` (Spaces), `参考数据` (Reference Data) → children `支付渠道` (Payment channels), `币种` (Currencies). Dashboard and Spaces are placeholder routes in Phase 1.

## Interaction & State Contract

> Per-screen behavior the executor must implement. Standard form/loading/error states.

**Navigation shell (D-12):**
- Persistent left sidebar (shadcn `sidebar`), collapsible on narrow viewports. Top-level groups: 仪表盘, 空间, 参考数据 (expandable to 支付渠道 / 币种).
- Active route highlighted with accent (background tint + icon), exactly one active item at a time.
- Each nav item has a lucide icon (e.g. `LayoutDashboard`, `Boxes`, `CreditCard`, `Coins`).

**Payment-channel screen (REF-01):**
- RSC-rendered table of active channels; "显示已归档" switch toggles inclusion of archived rows.
- Empty state shown when no active channels exist (and archived not shown).
- Add/Rename via a `dialog` with a single text field (React Hook Form + Zod). Submit button shows a pending/disabled state while the Server Action runs.
- Archive uses `alert-dialog` confirmation (copy above); reactivate is a direct action (no confirmation needed — it is additive).
- All mutations surface a sonner toast on success/failure; the table revalidates via `revalidatePath`.
- Loading: use `skeleton` rows for the initial table load if any client transition occurs; RSC first paint is server-rendered.

**Currency screen (REF-02):**
- Static read-only `table`, server-rendered. No actions, no dialogs.

**Accessibility:** all icon-only buttons carry `aria-label` / `tooltip`; dialogs trap focus (shadcn default); the show-archived switch is keyboard-operable; tables use proper `<th scope>` headers.
