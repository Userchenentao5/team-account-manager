<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser вҶ’ Server Action | channel name (FormData) is untrusted input crossing into the DB |
| Server Action вҶ’ SQLite | mutations on payment_channel |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-INPUT | Tampering | `addChannel` / `renameChannel` Server Actions | medium | mitigate | Re-validate every action's input with `channelSchema.parse` server-side (not just client RHF); Server Actions are public endpoints (ASVS V5) |
| T-03-MASS | Tampering | FormData вҶ’ channel insert | low | mitigate | Parse only the known `name` (and `id`) fields via Zod; never spread raw FormData into the insert (no mass-assignment) |
| T-03-SQLI | Tampering | channel queries/mutations | low | mitigate | Drizzle parameterized query builders only; no string-concatenated SQL |
| T-03-DEL | Tampering | channel removal path | low | mitigate | No hard-delete: all removals flip `is_active` (D-06); FK integrity for future space references preserved (D-07) |
</threat_model>
