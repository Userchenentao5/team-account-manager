# Phase 2 — UI Design Contract

> Visual and interaction contract for the Exchange-Rate Layer phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Scope this phase: a single **Reference Data → 汇率 (Rates)** screen — a 6-row rate table, a "rates as of `<date>`" label, a stale-data banner, and a manual "refresh rates" button. All tokens below are inherited from the Phase 1 design system (do NOT introduce new ones). Source: `src/app/globals.css`, `src/components/channels/channel-table.tsx`, `src/app/reference-data/currencies/page.tsx`, `src/components/nav/sidebar.tsx`.

## Color

Neutral (`baseColor: neutral`) radix-nova theme — there is no chromatic brand accent; the "accent" role is the near-black neutral `--primary`. Values are the OKLCH tokens from `src/app/globals.css` (`:root`).

| Role | Value (light) | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` `oklch(1 0 0)` (white) | Page surface, table background |
| Secondary (30%) | `--secondary` / `--muted` `oklch(0.97 0 0)`, `--sidebar` `oklch(0.985 0 0)`, `--card` white | Sidebar, muted/disabled row tint (`bg-muted/50`), table caption text (`text-muted-foreground`) |
| Accent (10%) | `--primary` `oklch(0.205 0 0)` (near-black) | See reserved-for list below |
| Destructive | `--destructive` `oklch(0.577 0.245 27.325)` (red) | Stale-data warning banner only (see Copywriting). There are **no destructive *actions*** this phase. |

Accent reserved for:
- The primary **"刷新汇率" (refresh) button** (the single primary CTA on the screen)
- The **active nav item** highlight in the sidebar (`data-[active=true]:bg-primary/10 data-[active=true]:text-primary`) for the new 汇率 entry — matches existing nav treatment

Accent is NOT used for: table borders, body text, badges, the "rates as of" label, or any decorative element. Status badges use `variant="secondary"` / `variant="outline"` (neutral), never accent.

## Copywriting Contract

All copy is Simplified Chinese (matches Phase 1 screens). Technical tokens (currency codes, dates) stay as-is.

| Element | Copy |
|---------|------|
| Primary CTA | `刷新汇率` (with `RefreshCw` icon, `size-4`; spinning while `isPending`) |
| "Rates as of" label | `汇率截至 {date}` — always shown when a cache exists, stale or not (D-05). `{date}` = formatted `fetched_at` (or `rate_date` if that optional column is chosen) |
| Table caption | `汇率以 1 单位外币折算为美元 (USD) 计;美元固定为 1。` |
| Stale banner (cache exists, refresh failed) | Heading `汇率可能已过期` · Body `无法连接汇率服务,当前显示上次缓存的汇率。请检查网络后重试"刷新汇率"。` (TriangleAlert icon, `destructive` Alert variant) |
| Empty state heading (no cache + first fetch failed) | `暂无汇率数据` |
| Empty state body | `尚未获取到汇率。点击"刷新汇率"从 Frankfurter 获取;若仍失败,请检查网络后稍后重试。` |
| Refresh success toast | `汇率已更新` (sonner `toast.success`) |
| Refresh failure toast | `刷新失败,已保留上次缓存的汇率。` (sonner `toast.error`) |
| Destructive confirmation | none — this phase has no destructive actions (no delete/clear). The destructive *color* is reused only for the stale warning banner. |

Interaction notes:
- Refresh button is **always available** (D-06), disabled only while its own transition is pending. Use `useTransition` + Server Action + `router.refresh()` — mirror `channel-table.tsx` exactly.
- Two distinct negative states (per RESEARCH Pitfall 5): **stale banner** when falling back to a *non-empty* cache; **empty state** when `rates.length === 0`. Never render a table of fake `0` rows. Never crash the page.
- Loading: optionally use the existing `Skeleton` component for the table body during the blocking lazy refresh; otherwise the RSC renders server-side and no client skeleton is needed.
