<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → Server Action (refreshRates) | Public endpoint; takes no client input — the untrusted input is the API response validated server-side in the FX service. |
| RSC → SQLite (read) | Server-side read of cached rates for display; native module forces Node runtime. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-02 | Denial of Service | rates/page.tsx + ensureFreshRates | medium | mitigate | Lazy refresh is time-bounded in the service; on failure the page renders cache + stale banner and never throws (FX-03 / Pitfall 5). |
| T-02-06 | Tampering | refreshRates Server Action | low | mitigate | No client input flows to the DB; trust boundary is the service-side Zod parse (Pitfall 6); revalidatePath only. |
| T-02-03 | Tampering | edge-runtime misconfig | low | mitigate | page/action stay on Node runtime (force-dynamic, no edge) — better-sqlite3 native module would crash on Edge (Pitfall 3). |
| T-02-SC | Tampering | shadcn `alert` add | low | accept | First-party shadcn official registry (components.json registries:{} — no third-party); copies component code, not an npm dependency. No package-manager install this phase, so no slopcheck/legitimacy checkpoint required. |
</threat_model>
