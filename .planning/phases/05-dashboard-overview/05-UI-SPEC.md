# Phase 5 - UI Design Contract

> Visual and interaction contract for the Dashboard & Overview phase. Generated by gsd-ui-researcher, pending verification by gsd-ui-checker.
>
> **Inheritance:** This phase replaces the placeholder root dashboard at `/` and must reuse the existing compact shadcn/Radix operational interface: sidebar shell, cards, tables, badges, tooltip-backed icon links, neutral tokens, `ExpiryBadge`, and monospaced money/date display. The dashboard is read-only and links into existing space detail/list flows; it must not introduce new mutation workflows.

**Phase 5 component inventory:**

| Component | Contract |
|-----------|----------|

| Metric summary grid | Four compact metric cards in this exact order: renewal-risk spaces, frozen USD cost total, total spaces, total child accounts. |

| Distribution summaries | Three compact bar-list cards: spend by country, by currency, and by payment channel. Use CSS bars and text totals, not a new chart library. |
| Count overview | Secondary count details after the expiring list and distributions. Include status counts and child-account counts by type if data is available. |
| Empty/error surfaces | Use the existing dashed empty-state pattern and restrained error copy. Never show fake zero totals as a loading fallback. |

## Spacing Scale

Declared values use the existing 8-point Tailwind scale and are all multiples of 4.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` | Badge/icon gaps, metric sublabel gaps, distribution row micro gaps |
| sm | 8px | `gap-2`, `p-2` | Inline links, table cells, compact bar labels, icon button clusters |
| md | 16px | `gap-4`, `p-4` | Metric card internals, distribution rows, mobile section stack |
| lg | 24px | `gap-6`, `p-6` | Page wrapper, top-level section gaps, card content padding |
| xl | 32px | `gap-8` | Desktop-only grid separation when distribution cards need breathing room |
| 2xl | 48px | `py-12` | Empty state vertical padding |
| 3xl | 64px | `p-16` | Avoid for this dashboard unless inherited from an existing primitive |

Mandatory page shell:
- Root wrapper remains `flex flex-col gap-6 p-6`.
- Header uses `flex flex-wrap items-start justify-between gap-4`.
- Metric grid uses `grid gap-4 sm:grid-cols-2 xl:grid-cols-4`.
- Expiring-space and distribution sections must start within the first viewport on a 1366x768 desktop after normal browser chrome. Do not insert decorative hero content above them.

Exceptions:
- Icon-only links use 44px touch targets (`className="size-11"`) with 16px lucide glyphs (`className="size-4"`), matching `SpaceTable`.
- Dashboard tables may use horizontal overflow with a stable `min-w-[720px]`; do not convert rows into tall marketing cards.
- Distribution bars use an 8px fixed height and must not resize layout when percentages change.

## Typography

Exactly four active sizes and two weights are allowed in this phase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Micro label | 12px | 400 regular | 1.4 | Metric captions, percentage labels, table helper lines |
| Body / table | 14px | 400 regular | 1.5 | Body copy, table cells, distribution row labels, empty-state body |
| Section heading | 16px | 600 semibold | 1.35 | Card titles, section headings, empty-state heading |
| Page heading / metric value | 24px | 600 semibold | 1.2 | Dashboard `h1`, primary metric numbers, top cost value |

Weights: regular (400) and semibold (600) only. Do not introduce bold display text or viewport-scaled type.

Numeric scan rules:
- USD amounts, original amounts, counts, percentages, dates, days remaining, currency codes, and subtotal lines use `font-mono`.
- Right-align numeric table columns.
- Keep metric labels short; wrap sublabels to a second line instead of shrinking text below 12px.

## Color

Use the existing neutral OKLCH palette from `globals.css`. This phase adds no brand colors, gradients, decorative blobs, or third-party chart palettes.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--foreground` | Page background, primary text, dashboard shell |
| Secondary (30%) | `--card`, `--muted`, `--muted-foreground`, `--secondary`, `--sidebar` | Metric cards, distribution cards, table hover, helper text, secondary links |
| Accent (10%) | `--primary` | Primary CTA only, active Dashboard sidebar state, selected/focused in-page anchor state |
| Destructive / critical | `--destructive` | Expired status badges and critical renewal-risk emphasis only; no dashboard destructive actions exist |
| Warning status | Existing `ExpiryBadge` amber `oklch(0.769 0.15 70)` | Soon-to-expire badge and matching small status marker only |

Accent reserved for:
- `查看到期空间` primary action when renewal-risk count is greater than zero.
- Active Dashboard sidebar/nav state inherited from `AppSidebar`.
- Focus ring and selected in-page anchor state only.

Status display rules:
- Expired spaces use `ExpiryBadge` destructive styling and may add a left border in `border-destructive`.
- Soon-to-expire spaces use the existing amber badge only; do not invent a second warning palette.
- Normal and empty states use outline/secondary variants.
- Distribution bars use neutral chart tokens `--chart-1` through `--chart-5` or `--primary` at low opacity; exact text totals remain the primary data carrier.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Route and rendering | `/` is a read-oriented Server Component backed by aggregate query helpers. Keep `force-dynamic` if needed for SQLite/native module access. |
| First-screen priority | The first viewport must show the dashboard heading, all four metric cards, and the start of the expiring-space list. Distribution sections are secondary. |
| Metric card order | Use this exact order: `到期风险空间`, `冻结 USD 成本合计`, `空间总数`, `子账号总数`. Do not reorder by implementation convenience. |
| Renewal-risk metric | Count only expired plus soon-to-expire spaces. If count is greater than zero, show `查看到期空间` as the only primary CTA and anchor/link it to the expiring-space section. |
| Cost metric semantics | Use stored USD minor-unit values only: `space.amountUsd` plus `childAccount.monthlyAmountUsd` when the dashboard cost footprint includes child monthly cost. Do not recompute from live FX rates. |
| Cost reconciliation | Display the grand total and the two subtotals `空间付款 USD` and `子账号月度 USD`. The subtotals must sum exactly, in integer minor units, to the grand total shown. |
| Expiring list order | Sort expired spaces first, then soon-to-expire spaces, then by `expiryDate` ascending. Do not show normal spaces in this section. |
| Expiring list columns | Use: status, space name, expiry date, days remaining/overdue, frozen USD cost, child-account count, payment channel, action. |
| Expiring row action | Space name and `Eye` icon link both go to `/spaces/[id]`. Tooltip and aria-label use `查看空间详情`. No inline edit/delete controls. |
| Empty dashboard | If there are zero spaces, show one dashed empty state after the page heading. Do not show four zero metric cards above it. |
| No renewal risk | Keep all metric and distribution content visible. In the expiring section, show `暂无到期风险` empty copy instead of hiding the section. |
| Distribution summaries | Use three bar-list cards: country, currency, payment channel. Each row shows label, USD total, percentage of grand total, and a fixed-height bar. |
| Distribution links | Country and payment-channel rows may link to existing `/spaces` filters. Currency rows remain text-only unless a currency filter already exists. |
| Count overview | Show total spaces, expired spaces, soon spaces, normal spaces, child accounts, and child accounts by `seatType` when available. Keep this secondary to renewal risk and spend. |
| Loading state | Prefer server-rendered data. If suspense is introduced, use existing `Skeleton` primitives and never render fake zeros. |
| Error state | Use the error copy above in a compact card/alert. Do not expose stack traces or database paths. |

Responsive rules:
- Desktop: metric grid is 4 columns; expiring list gets full width; distributions use a 3-column grid when space allows.
- Tablet: metric grid is 2 columns; distributions can be 2 columns plus wrap.
- Mobile: metric grid is 1 column; tables sit in horizontal overflow containers; no text overlaps buttons or bars.
