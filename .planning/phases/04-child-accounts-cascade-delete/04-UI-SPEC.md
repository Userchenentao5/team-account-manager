# Phase 4 - UI Design Contract

> Visual and interaction contract for the Child Accounts & Cascade Delete phase. Generated by gsd-ui-researcher, pending verification by gsd-ui-checker.
>
> **Inheritance:** This phase extends the approved Phase 3 space detail experience. New UI must stay inside the existing space detail page and reuse the project's shadcn/Radix primitives, table row actions, dialog forms, alert-dialog confirmations, lucide icon buttons, and sonner toast feedback. Do not add a global child-account page, sidebar entry, or credential fields.

## Spacing Scale

Declared values use the existing 8-point Tailwind scale and are all multiples of 4.

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1` | Table row action icon cluster gaps; badge/icon gaps |
| sm | 8px | `gap-2`, `p-2` | Inline controls, tooltip/action groups, table cells |
| md | 16px | `gap-4`, `p-4` | Dialog form fields, card content stack, alert-dialog body |
| lg | 24px | `gap-6`, `p-6` | Page wrapper, space detail section stack, top-level header |
| xl | 32px | `gap-8` | Reserve for responsive two-column form grouping only when needed |
| 2xl | 48px | `py-12` | Empty state vertical padding |
| 3xl | 64px | `p-16` | Rare page-level spacing; avoid in dense management views |

Mandatory page shell: keep the detail page wrapper as `flex flex-col gap-6 p-6`. Keep header actions in a wrapping flex row with `gap-2` inside the action cluster and `gap-4` between title and controls.

Exceptions:
- Icon-only row actions use 44px touch targets (`className="size-11"`) with 16px lucide glyphs (`className="size-4"`), matching the existing space table pattern.
- Dialog widths may use existing component constraints: child add/edit dialog `sm:max-w-2xl`; destructive confirmation dialogs `sm:max-w-sm` or the alert-dialog default.
- Table cells keep existing `p-2`; do not enlarge child rows into card-like rows.

## Typography

Exactly four active sizes and two weights are allowed in this phase.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Page heading | 24px | 600 semibold | 1.2 | Space detail `h1`, unchanged from Phase 3 |
| Section heading | 16px | 600 semibold | 1.35 | Card titles, dialog titles, alert-dialog titles |
| Body / table / labels | 14px | 400 regular | 1.5 | Table cells, descriptions, form labels, helper text |
| Form input base | 16px mobile / 14px desktop via existing `Input` | 400 regular | 1.5 | Inputs and selects; keep existing shadcn control sizing |

Weights: regular (400) and semibold (600) only. Do not introduce bold display text.

Numeric and identifier rule:
- Money amounts, frozen USD, current CNY estimate, dates, monthly payment day, currency codes, and rate-as-of values use `font-mono`.
- Right-align numeric money columns in child account tables.
- Email/login values may use normal text in cards, but use `font-mono` in table cells when it improves scanability.

## Color

Use the existing neutral OKLCH palette from `globals.css`. This phase adds no brand colors and no decorative gradients.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--foreground` | Page background, table surface, primary text |
| Secondary (30%) | `--card`, `--muted`, `--muted-foreground`, `--secondary`, `--sidebar` | Cards, sidebar, table hover, helper text, secondary buttons |
| Accent (10%) | `--primary` | Primary CTA only, active sidebar/nav state if already present, checked switch for `can_change_seat_type` |
| Destructive | `--destructive` | Child delete, space cascade delete, destructive warnings, destructive toasts/errors only |

Accent reserved for:
- `新增子账号` primary button.
- `保存子账号` submit button in child add/edit dialogs and `保存母账号席位` submit button in mother seat dialogs.
- Checked state of the mother-account `can_change_seat_type` switch.
- Existing active sidebar/nav state only. Do not add a child-account nav item.

Semantic display rules:
- Seat type badges use neutral variants only: `codex` = `Badge variant="secondary"`, `chatgpt` = `Badge variant="outline"`. Do not use saturated color to distinguish products.
- Frozen USD is the authoritative amount and should use primary foreground text.
- Current CNY estimate is secondary display text (`text-muted-foreground` plus `font-mono`) and must never look more important than USD.
- Destructive controls must use `variant="destructive"` only for irreversible child delete and space cascade delete confirmations.

## Interaction Contract

| Interaction | Contract |
|-------------|----------|
| Child account placement | Child accounts are managed only on `/spaces/[id]` inside the existing space detail page. No sidebar item, no `/child-accounts` route, no global list. |
| Space detail visual focal point | On `/spaces/[id]`, the primary visual anchor is the child account table section. `新增子账号` is the only primary button in that section. |
| Child account table | Use existing `Table` primitives with columns: type, email/login, note/label, joined date, monthly price, frozen monthly USD, monthly payment day, actions. |
| Child row actions | Use lucide icon buttons with tooltips: `Pencil` for edit and `Trash2` for delete. Buttons are `variant="ghost"`, `size="icon"`, `className="size-11"`, and include `aria-label`. |
| Add/edit child | Open a dialog from `新增子账号` or row edit. Use `react-hook-form`, `zodResolver`, server-side re-parse, `useTransition`, inline field errors, and sonner toasts. |
| Child monthly price | Currency is selected per child. Original monthly amount and currency are editable. Frozen USD monthly price is computed on save and displayed read-only after save. |
| FX no-rate state | Block save for non-USD currencies without a cached rate. Show inline form error and `toast.error` with the no-rate copy above. |
| Edit child refreeze | Editing email/login, note/label, joined date, seat type, or payment day must not visually imply historical USD changes. If amount/currency changes, save may refreeze monthly USD; otherwise display the prior frozen value. |
| Mother seat edit | Provide editable `seat_type` select (`codex` / `chatgpt`) and `can_change_seat_type` switch. Child accounts must not show this switch. |
| Space delete | Put delete action in the existing detail action area or a clearly separated destructive zone on the detail page. It must not appear on the space list in this phase unless the planner explicitly scopes it. |
| Cascade delete dialog | Alert dialog shows irreversible scope, child count if available, exact-name input, disabled confirm until exact match, destructive confirm button, cancel button first on mobile via existing footer behavior. |
| Current CNY estimate | Display next to frozen USD in the space detail cost section. Label as current/reference display, not stored or frozen. If unavailable, show `暂无 CNY 参考` and keep frozen USD visible. |
| Toast feedback | Success and failure feedback must use `sonner`. Success examples: `已新增子账号`, `已保存修改`, `已删除子账号`, `已删除空间`. |
| Refresh/navigation | After mutations, refresh current detail page and `/spaces` list data via existing Server Action revalidation pattern. |

Seat type display:

| Value | Display |
|-------|---------|
| `codex` | Badge text `codex`, neutral secondary badge |
| `chatgpt` | Badge text `chatgpt`, neutral outline badge |

Child form field order:
1. Seat type (`codex` / `chatgpt`)
2. Email/login
3. Note/label
4. Joined date
5. Monthly amount
6. Monthly currency
7. Monthly payment day

Do not include frozen USD fields as editable inputs. Show frozen USD in table/detail display only after save.
