<objective>
Replace ECS source synchronization and server-side image builds with a private GHCR image workflow. Preserve SQLite data and restore a verified database backup when a replacement container fails.
</objective>

<tasks>

<files>.github/workflows/deploy.yml, Dockerfile</files>
<action>Build the production image once after CI, publish it to private GHCR with the commit SHA and deploy the returned immutable digest.</action>
<verify>GitHub Actions Verify and Build image jobs complete successfully.</verify>
<done>The deployment job consumes a GHCR digest and does not build application source on ECS.</done>
</task>

<task type="auto">
<name>Task 2: Deploy a candidate Compose file with rollback</name>
<files>compose.yaml, scripts/deploy-production.sh</files>
<action>Pre-pull the private image using an ephemeral registry login, snapshot SQLite through better-sqlite3, start the candidate Compose service, and restore the prior image and snapshot on failure.</action>
<verify>bash -n scripts/deploy-production.sh</verify>
<done>A failed container replacement leaves the previous image and database backup restorable without an ECS source build.</done>
</task>
</tasks>
