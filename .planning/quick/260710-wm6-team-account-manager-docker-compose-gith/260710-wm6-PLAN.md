<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions runner to ECS SSH | CI source and deployment credentials cross the public network to the server. |
| Repository source to ECS deployment directory | A verified main commit replaces executable application files but must not replace persistent state. |
| Public HTTPS proxy to loopback app | Internet traffic terminates at the separately configured TLS proxy before reaching the app. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-CICD-01 | Spoofing | SSH connection | high | mitigate | Require a manually verified ECS_SSH_KNOWN_HOSTS secret and StrictHostKeyChecking=yes; CI must not trust a runtime key scan. |
| T-CICD-02 | Information disclosure | GitHub workflow and image context | high | mitigate | Use GitHub Secrets in mode-600 files, redact no secrets into logs, and exclude .env files and data from rsync and Docker context. |
| T-CICD-03 | Tampering | CI artifact and remote source | high | mitigate | Deploy only after verify, check out github.sha in both jobs, pin Actions by immutable SHA, and use rsync from that checkout rather than a remote mutable branch. |
| T-CICD-04 | Data loss | SQLite database and environment file | high | mitigate | Bind-mount ./data, keep .env.production server-only, and exclude both from rsync deletion and image build context. |
| T-CICD-05 | Information disclosure | Public application listener | high | mitigate | Bind Compose only to 127.0.0.1:3000 and require ECS HTTPS reverse-proxy/firewall setup before public access. |
| T-CICD-06 | Elevation of privilege | ECS deploy account with Docker access | high | accept | Docker access is required for the requested deployment; use a dedicated key-only account, restrict its key, and keep its private key solely in GitHub Secrets. |
</threat_model>
